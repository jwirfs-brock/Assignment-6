Design updates

For this project, I added a low-vision mode, which displays all of the content in a bigger font and displays light-on-dark text.

I went with the alternate index.html solution, which I actually don't think is ideal. The problem with this is that when I toggle the formats, it refreshes the state of the DOM, so I lose whatever tab I was on at the moment. So if I toggle the large format mode from the "Add Contact" tab, it goes back to the "Dialer" tab. I tried (pretty hard, atually!) to implement a solution that just toggles back and forth between the style sheets, but I couldn't get it to work. I think that's ultimately a better solution because then your page can remember the DOM state, so the tab navigation will be more fluid.

Lessons learned using a magnifier

I played around with the built in MacOSX magnifier features. Here are some things I observed/learned:

  • Scrolling sometimes happens very quickly when you are zoomed in! You have to be careful with this (or change your scroll speed).
  • With the MacOSX magnifier, it's easy to toggle between max zoom and regular zoom, but not easy to get intermittent zoom levels. To do this, it's much easier to use the browser zoom functions.
  • I tended to go back and forth a lot between the zoom mode and the regular mode, using the regular mode for large-scale navigation around page regions, and then the zoom mode for interactive elements (entering text into a field, selecting a check-box, etc.).
  • I interacted with a map, and noticed that the magnifier started to break down because the map had its own zoom controls (for the map content). When I was zoomed in on an area of interest within the map, using the magnifier, I could't see the map zoom controls.
  • I also played with changing the size of my cursor, and really enjoyed that feature! I think I'm actually going to stick with an enlarged cursor from here on out.
